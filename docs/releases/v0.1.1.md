# otel-policy-lab v0.1.1

**Hardening release:** CI smoke coverage for the GitHub Action, toolchain pinning improvements, explicit CLI entrypoint, and expanded report-redaction tests.

## Highlights

- GitHub Action reads Go version from `go.mod` via `go-version-file` (removed `go-version` input)
- New `action-smoke` CI job mirrors composite-action build and runs the passing example
- Explicit `cmd/otel-policy-lab/main.go` entrypoint with injectable version string
- Additional tests proving JSON reports never leak secrets on resource-level failures
- Expanded ecosystem comparison docs and failure-mode guidance

## Changes

### GitHub Action

- **Changed:** `actions/setup-go` now uses `go-version-file: ${{ github.action_path }}/go.mod` instead of a `go-version` input defaulting to `1.22.4`
- **Removed input:** `go-version` — consumers no longer need to pin Go separately; the action follows the repo's `go.mod`

**Migration:** If you passed `go-version` explicitly, remove it. The action inherits the toolchain from `go.mod`.

```yaml
# Before (v0.1.0)
- uses: andyphied/otel-policy-lab@v0.1.0
  with:
    go-version: "1.22.4"
    collector-config: ./collector.yaml
    ...

# After (v0.1.1)
- uses: andyphied/otel-policy-lab@v0.1.1
  with:
    collector-config: ./collector.yaml
    ...
```

### Tooling

- Added `.golangci.yml` and golangci-lint job in CI
- Added `action-smoke` job: builds the binary the same way the composite action does and runs the passing example
- Composite action build sets `CGO_ENABLED=0` and `GOTOOLCHAIN=local`

### Code

- Added `cmd/otel-policy-lab/main.go` as the canonical CLI entrypoint (`version` injected at link time)

### Tests

- Secret-redaction fixture for end-to-end JSON report validation
- Tests for forbidden resource values and required-resource failures across traces and metrics (no raw secrets in report JSON)
- Traces-only and metrics-only OTLP parse coverage
- Terminal output test handles pipe errors explicitly

### Documentation

- Expanded [`docs/comparison-to-existing-tools.md`](../comparison-to-existing-tools.md) with config-level vs output-level comparison
- Documented vacuous passes on empty signals and fixture runner label deletion in design/failure-mode docs
- README reframed around failure modes

## Upgrade notes

- Pin `@v0.1.1` in workflows; remove any `go-version` action input
- No policy file or CLI flag changes in this release
- Behavior of policy evaluation is unchanged from v0.1.0

## GitHub Action (v0.1.1)

```yaml
steps:
  - uses: actions/checkout@v4
  - uses: andyphied/otel-policy-lab@v0.1.1
    with:
      collector-config: ./collector.yaml
      input: ./fixtures/checkout.otlp.json
      policy: ./policy.yaml
      report: ./otel-policy-report.json
```
